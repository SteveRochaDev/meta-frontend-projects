## Introduction
Semantic HTML plays a crucial role in creating accessible, readable, and maintainable web pages. By using meaningful tags such as `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, and `<footer>`, developers provide structure and context to web content that benefits both users and assistive technologies.

This lab focuses on refactoring a non-semantic webpage into a semantic one without changing its content or visual layout.

## Goal
The goal of this lab is to transform a non-semantic HTML webpage into a well-structured semantic webpage while preserving the original layout and content.

## Objectives
- Understand the purpose of semantic HTML elements.
- Identify non-semantic structures in an HTML document.
- Replace generic elements with appropriate semantic tags.
- Maintain layout consistency while improving accessibility and structure.

## Semantic Elements Used
- `<header>` for the page header.
- `<nav>` for the navigation menu.
- `<main>` for the main content area.
- `<section>` to group related content.
- `<article>` for individual blog posts.
- `<footer>` for contact information.

## Key Takeaways
- Semantic HTML improves accessibility and SEO.
- Proper structure makes code easier to read and maintain.
- Semantic refactoring does not require visual changes.
- Using meaningful tags is a best practice in modern web development.
