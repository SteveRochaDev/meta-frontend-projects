# Lab: Create and Test a Form

## Introduction
Forms are a fundamental part of web development, enabling user interaction and data submission. HTML provides built-in validation features that help ensure data quality and improve the user experience without requiring JavaScript.

In this lab, a basic login form is created for the Little Lemon website using native HTML form validation.

## Goal
The goal of this lab is to build and test a login form with required fields and minimum input length validation using standard HTML attributes.

## Objectives
- Create an HTML form inside the main content area.
- Add username and password input fields.
- Use semantic labels associated with inputs.
- Apply required and minlength validation attributes.
- Ensure secure password input using the correct input type.
- Test form behavior using browser validation.

## Form Features
- Username field with minimum length validation.
- Password field with hidden input.
- Required fields enforced by HTML validation.
- Submit button to trigger form submission.
- Visual feedback for invalid input using CSS.

## Key Takeaways
- HTML forms allow data collection without JavaScript.
- Built-in validation improves usability and accessibility.
- Labels enhance form accessibility and user experience.
- Proper input types protect sensitive information.
