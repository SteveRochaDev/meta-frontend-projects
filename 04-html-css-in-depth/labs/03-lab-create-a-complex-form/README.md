# Lab: Create a Complex Form

## Introduction
Forms allow users to interact with web applications by submitting structured data. HTML provides several input types that enhance usability and validation, such as email, date, number, and checkbox inputs.

This lab focuses on building a more complex form for table reservations using native HTML features.

## Goal
The goal of this lab is to create a table booking form that collects user information and applies appropriate client-side validation.

## Objectives
- Create a form inside the main content area.
- Add multiple input fields with different input types.
- Use HTML validation for email, numeric ranges, and required fields.
- Include a required checkbox to confirm agreement with a policy.
- Test form behavior using browser validation.

## Form Features
- Email address field with client-side validation.
- Date picker for booking date.
- Numeric input for number of people with min and max limits.
- Required checkbox for cancellation policy agreement.
- Submit button to complete the reservation.

## Key Takeaways
- HTML input types improve data accuracy and user experience.
- Client-side validation reduces invalid submissions.
- Labels improve accessibility and usability.
- Complex forms can be built without JavaScript.
