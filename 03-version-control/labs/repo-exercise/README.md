## Introduction
This lab focuses on practicing essential Git and GitHub workflows using the command line and GitHub CLI. The exercise covers repository creation, authentication, cloning, committing changes, and pushing updates to a remote repository.

## Objectives
- Create a GitHub repository
- Authenticate using GitHub CLI
- Clone a remote repository
- Track and commit files using Git
- Push changes to GitHub

## Steps Completed

1. Created a new GitHub repository named **repo-exercise** with a README file.
2. Authenticated GitHub CLI using `gh auth login`.
3. Generated and configured an authentication token with the required scopes.
4. Cloned the repository locally using: gh repo clone <username>/repo-exercise
5. Navigated into the repository directory.
6. Added a new file named `result.txt`.
7. Verified file status using `git status`.
8. Tracked the file using `git add result.txt`.
9. Committed the file with the message: Successful exercise
10. Pushed the commit to the remote repository.
11. Verified that the file appears on GitHub.

## Files Included
- `README.md`
- `result.txt`

## Key Takeaways
- GitHub CLI simplifies authentication and repository management.
- Git status helps track file changes clearly.
- Committing and pushing changes ensures synchronization with remote repositories.
- Version control is essential for collaborative and professional development workflows.
