# Lab: Create and Manipulate Directories and Files

## Introduction
Organizing files and directories efficiently is a core skill for developers. The command line provides powerful tools to create, move, and inspect files and directory structures, making project setup and file management faster and more reliable.

In this lab, basic command-line commands are used to build and manipulate a structured directory hierarchy.

## Goal
The goal of this lab is to create and manage directories and files using command-line commands, including moving files between directories and inspecting directory contents.

## Objectives
- Create a working directory and subdirectories.
- Create files using the command line.
- Move files between directories.
- Create nested directory structures.
- Inspect directory contents using listing commands.

## Commands Practiced
- `mkdir` to create directories.
- `mkdir -p` to create nested directories.
- `touch` to create files.
- `mv` to move files between directories.
- `cd` to navigate the file system.
- `ls -l` to inspect directory contents.
- `pwd` to verify the current working directory.

## Key Takeaways
- Directory structures help keep projects organized and maintainable.
- Command-line tools allow fast and precise file management.
- Understanding relative paths is essential when moving between directories.
- Inspecting directories helps verify file locations and structure.
